A few are enterprise whales, but the majority are students, personal emails, or B2C noise.\n\nEnterprise Revenue Operations (RevOps) teams buy tools like **Clearbit Reveal**, **MadKudu**, or **ZoomInfo** to enrich these emails, score them, and route the good ones to Sales. \n\n**The catch?** These tools cost upwards of **$20,000/year** and often operate as complete "black boxes" (nobody knows exactly *why* a lead got 90 points).\n\n## The Solution: How It Works\n\n**LeadGenius** offers a transparent, $0/month open-source alternative:\n\n1. **Ingest (FastAPI)**: A lightweight webhook endpoint catches incoming leads from your frontend or Zapier.\n2. **Enrich (The Waterfall)**: Combines standard firmographic APIs (Apollo.io/Hunter.io) with a fallback **LLM Web Scraper**. If standard APIs fail, LeadGenius visits the lead\'s website and uses AI (OpenAI/Anthropic via OpenRouter) to read the homepage and extract precise B2B intent.\n3. **Score (dbt + Postgres)**: Uses **Configuration-as-Code** (a simple `rules.yml` file) to calculate Lead Scores via dbt SQL. 100% transparent. 100% customized to your ICP.\n4. **Activate & Route (Reverse ETL)**: An internal orchestrator queries the database for "Hot" leads, alerts your team on Slack, and syncs the exact score back into your CRM.\n\n## Key Features\n\n*   🌊 **Waterfall Enrichment**: Never pay for empty API pings. Tries cheap APIs first, falls back to intelligent LLM Scraping.\n*   🧠 **LLM Web Scraper Module**: Built-in BeautifulSoup + OpenAI integration. Analyzes a company\'s homepage to extract hidden data (Industry, B2B/B2C, Audience).\n*   ⚙️ **Transparent YAML Scoring**: Define what matters to you (e.g., `role: CEO -> +50 points`) in plain English. LeadGenius dynamically generates the SQL.\n*   📊 **Evidence BI Dashboard Ready**: Includes an automated GitHub Actions pipeline that generates a beautiful RevOps dashboard (Funnel, Missing Data Fill Rates, Score Histograms) every night.\n\n## CRM Integrations\n\nLeadGenius doesn\'t just alert you; it actively patches your existing workspace:\n\n*   **HubSpot**: Includes a native `HubSpotDestination` module. It uses the `HUBSPOT_API_KEY` to search the HubSpot Contacts API by email and perform a `PATCH` request to update custom properties (`leadgenius_score`, `leadgenius_tier`).\n*   **Slack**: Real-time "Hot Lead" alerts in a dedicated channel.\n*   [Extendable] Modular `ActivationManager` makes writing a new Salesforce or Pipedrive integration a 10-line python class.\n\n## Quick Start\n\n### Prerequisites\n*   Docker & Docker Compose\n*   Python 3.11+\n\n### Setup\n\n```bash\n# 1. Clone the repo\ngit clone https://github.com/Astoriel/Job-Bodyguard-1.git leadgenius\ncd leadgenius\n\n# 2. Configure environment\ncp .env.example .env\n# Fill in your OpenRouter / Apollo / HubSpot / Slack keys\n\n# 3. Define your rules\nnano rules.yml\n\n# 4. Spin up the cluster (Postgres + FastAPI + Scheduler)\ndocker-compose up -d --build\n```\n\n### Ingesting a Lead\n\nSend a POST request to your new engine:\n```bash\ncurl -X POST http://localhost:8000/webhook/leads \\\n  -H "Content-Type: application/json" \\\n  -d \'{"email":"sam@openai.com", "name":"Sam Altman", "source":"homepage"}\'\n```\n\nWatch the terminal (`docker-compose logs -f api`) to see the engine execute the Waterfall, trigger `dbt run`, and route the lead!\n\n## Built With\n\nLeadGenius relies heavily on the modern data and backend stack:\n\n*   **Python 3.11** & **FastAPI** - Core Server & Webhooks\n*   **dbt (Data Build Tool)** - SQL-based Data Transformations & Scoring models\n*   **PostgreSQL** - Central Data Warehouse\n*   **Evidence BI** - RevOps Dashboard Visualization\n*   **SQLAlchemy** & **Pydantic** - ORM and Type Validation\n*   **APScheduler** - Background Task / CRON Management\n*   **OpenRouter** / **OpenAI SDK** - LLM Fallback Web Scraping\n*   **GitHub Actions** - CI/CD & Automated Mock Data Deployments
